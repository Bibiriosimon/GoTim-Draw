1. 题目批改与图解讲题

视觉结构

推荐采用“原题层 + 重构图 + 推导区 + 结论区”。原题图保留在左侧或上方，AI 在图上圈出已知条件，再用可编辑图形重新画出干净示意图。

  • 左侧：题目截图、条件高亮、箭头指向关键量。
  • 中间：重新绘制的几何/物理示意图。
  • 右侧：公式、推导、易错点。
  • 底部：函数图、答案、单位检查。

Python 的角色

Python 适合做精确计算，不适合作为最终静态截图输出。让 Python 计算采样点、极值、交点、坐标，再转换为画板命令。

Python/SymPy/NumPy
-> points, labels, extrema
-> createDraw / createLine / createText
-> editable canvas result
示例：带电环轴线上电场。AI 可以推导 E(x)=kQx/(x^2+R^2)^(3/2)，用 Python 找到最大值 x=R/sqrt(2)，再在画板上画环、轴线、P 点、公式区和红色函数曲线。
公式书写规则：公式不要用普通 createText 写成 sqrt(...)、R^2、R^3 或很长的一整行推导。AI Coding 应优先使用 createFormula，例如 v_1 = sqrt(GM/R) 会在画板中规范成更易读的 v₁ = √(GM/R)，M/R^3 -> GM/R = G*(M/R^3)*R^2 会规范成带上标和箭头的形式。长推导应拆成多行。

4. 棋类与现场编程扩展

五子棋当前能力

  • 15 x 15 交叉点棋盘。
  • 用户点击落子，AI 自动回应。
  • 横、竖、斜五连胜负判断。
  • 画板同步绘制棋盘、棋子和胜利线。

新棋类工作流

  1. 搜索或确认规则。
  2. 定义棋盘、棋子、合法走法、胜负条件。
  3. 先实现最小规则引擎。
  4. 再实现画板 UI 和策略讲解。
  5. 把规则写入 /agent.md 和 /api/capabilities。

5. 用户手绘识别到后端 Agent

未来画板不仅由 Agent 绘制，也可以理解用户手绘。当前前端已经会向后端同步 shape summary，这是手绘识别的入口。

阶段做法应用
当前GET /api/canvas/shapes 读取画布对象、文字、边界、points。让 Agent 理解现有画布。
下一步加入选区/最近手绘 shape 分析。定点讲题、局部修改。
进阶识别手绘箭头、圈选、函数草图、轮廓。画画比赛、AI 纠错、协作绘画。

性能策略：不要逐点实时传输用户指针事件。使用停笔 debounce、选区分析或手动触发识别。
